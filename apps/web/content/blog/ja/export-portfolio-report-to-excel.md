**ポートフォリオ分析の本物のExcelワークブック作成は、以前はティッカーと価格を引き出す1時間、指標をクロスリファレンスする1時間、他人が使えるよう出力をフォーマットする3時間目を意味しました。** このチュートリアルは同じワークフローを60秒で示します — 全セルがライブデータから出典、全数式が本物、各シートがアナリストが実際にワークブックを共有する方法で構造化。ダウンロードした`.xlsx`は動く作業ファイル:開いて、任意の列をソートし、ピボットを構築、同僚と共有。何もスクリーンショットではなく、何もフラットなダンプではない。

これは4ステップのチュートリアル。各ステップはプロンプト1つかクリック1つ。PickSkillでポートフォリオが設定済みなら、全フローを1分以内に実行できます。

### 重要なポイント

- **4ステップ、約60秒。** ポートフォリオを開く、エクスポートをクリック、Excelを選ぶ、お好みのスプレッドシートでリフレッシュ。
- **全値がライブデータから出典** — 価格は市場フィードから、指標は最新終値で計算、財務は最新ファイリングから。
- **ワークブックは複数シートでフィルタリング用に構造化** — Holdings、Indicators、Signal Trail、Valuation、Trade Logプレースホルダー。
- **Excel、Google Sheets、LibreOffice Calc、Numbersと互換。** 全体でOpenXMLフォーマット — プラットフォームロックインなし。
- **米国・香港・A株保有銘柄で動作**、シートごとに適切な市場慣習。

## なぜこれが重要か

Excelはポートフォリオ分析の普遍的交換フォーマットのまま。PDFは読み取り専用、プレゼンデッキはプレゼン形状、チャットスレッドはソートできない。Excelワークブックは、協調的に*作業*できる唯一のフォーマット — ピボットを構築、列を追加、自分のデータとクロスリファレンス。

PickSkillはワークブック組み立てステップを1クリックに圧縮するので、時間が分析に戻ります。3つの即時ユースケース:

- **個人のポートフォリオダッシュボード。** ワークブックを保存し、毎週再エクスポートすれば、指標がどう進化したかのローリングトレイルを得る。
- **協力者との共有。** 友人、投資クラブ、パートナー — Excelを持つ誰でも同じファイルを開いて貢献できる。
- **独自のカスタム分析を上に構築。** エクスポートワークブックは出発点;ベースを再構築せずに独自の列、シナリオ、ノートを重ねる。

## 4ステップワークフロー

### ステップ1 — エクスポートしたいポートフォリオを開く

[/portfolios](/portfolios)へ移動。ワークブックにしたいポートフォリオを選ぶ。(初回設定は[テクニカル指標でポートフォリオを追跡](/blog/track-a-portfolio-with-indicators)を参照。)

ワークブックはポートフォリオサイズ全体でうまくスケール — 3銘柄ウォッチリストから50銘柄分散ブックまで。より大きなポートフォリオはより厚いIndicatorsシートを生むが、構造は一貫したまま。

### ステップ2 — 「Export to Excel」をクリック

ポートフォリオ詳細ページのヘッダーには「Export to chat」ボタンのクラスターがあります。Excelボタンをクリック。PickSkillはポートフォリオ文脈を含む事前入力されたプロンプトでチャットを開きます。

デフォルトプロンプトは5シートワークブックを生成。送信前にカスタマイズするには:

- **特定の指標を追加または他を省略**:「MACD、RSI、MAスタックのみ含める — ボリンジャーとKDJをスキップ」。聴衆が特定の指標に不慣れなときに有用。
- **ファンダメンタルを追加**:「保有銘柄ごとに直近4四半期の売上とEPSを含める」。ファンダメンタルシートがワークブックに現れる。
- **シナリオを追加**:「保有銘柄ごとに強気/ベース/弱気の価格目標を持つシナリオシートを追加」。記入用の空白シナリオシートが追加される。

### ステップ3 — PickSkillがワークブックを組み立てる約30秒待つ

PickSkillは順番に以下を行います:

1. 全保有銘柄について現在価格、日中メトリクス、過去価格系列(デフォルト6か月)を取得。
2. 完全な指標スイート([MACD](/blog/what-is-macd)、[RSI](/blog/what-is-rsi)、[KDJ](/blog/what-is-kdj)、[ボリンジャーバンド](/blog/what-is-bollinger-bands)、[ADX](/blog/what-is-adx)、MAスタック、[出来高](/blog/what-is-volume-analysis)、[資金フロー](/blog/what-is-capital-flow))を実行。
3. 次元ごとの現在シグナル状態と5日バケットトレイルを検出。
4. 評価倍率と比率(P/E、P/B、EV/EBITDA、配当利回り)を取得。
5. 全値を構造化された複数シートワークブックにレンダリング。
6. シート間でクロスリファレンスするExcel数式を追加するので変更が伝播。
7. 結果を埋め込みスパークラインと条件付きフォーマット付きの`.xlsx`ファイルに書き込み。

作業のストリーミングサマリーが表示されます。完了すると、7日間有効なダウンロードリンクを取得。

### ステップ4 — Excelで開いてカスタマイズ

ダウンロードした`.xlsx`は本物の動く作業ファイル。任意のスプレッドシートツールで開く。よくある即時編集:

- **シグナル強度でソート** — バケットスコアフィールドの列ヘッダーをクリック。
- **ノート列を追加** — 全シートには個人注釈用に右側にノート列がある。
- **ピボットテーブルを構築** — データの異なる切り取り方のために、セクターまたはシグナルバケットで保有銘柄をピボット。
- **独自の列を追加** — ワークブックは拡張用に設計;フィールドを追加しても何も壊れない。

より大きな編集には、チャットに戻り特定の更新をリクエスト:

```text
各保有銘柄の1か月、3か月、6か月パフォーマンスを
SPYベンチマークに対して示すシートを追加。
```

```text
indicatorsシートをヒートマップカラースキームで再作成 — 強気バケットに緑、
弱気に赤、ニュートラルに琥珀。
```

```text
MACD、RSI、KDJすべてで同時にアクティブダイバージェンスを持つ
保有銘柄をすべてフラグするスクリーンシートを追加。
```

PickSkillは新しい構造でワークブック組み立てを再実行し、フレッシュなダウンロードを提供。

> **今すぐ試す。** [/portfoliosへ](/portfolios)、任意のポートフォリオに入り、「Export to Excel」をクリック。全ループは1分以内。

## 出力はどう見えるか

デフォルト5シート構造:

| シート | コンテンツ |
|---|---|
| **1. Holdings** | 保有銘柄ごとに1行 — ティッカー、名称、市場、ウェイト、現在価格、日次変化、5日変化、1か月変化、ポジション価値、ノート。ソート可能。 |
| **2. Indicators** | (保有銘柄×指標)ごとに1行 — 現在値、バケットラベル、5日バケットトレイル、シグナル注釈。Holdingsシートへのクロスリファレンス。 |
| **3. Signal Trail** | 保有銘柄ごとに1行 — 8つの指標次元すべてにわたる完全な5日進化、バケット遷移がハイライト。 |
| **4. Valuation** | 保有銘柄ごとに1行 — P/E、フォワードP/E、P/B、EV/EBITDA、配当利回り、比較用のセクター中央値。 |
| **5. Trade Log** | 事前フォーマット済み空白シート — 日付、ティッカー、アクション、数量、価格、根拠の列。独自のトレードジャーナリング用、保有銘柄リストが事前入力済み。 |

シートはリンクされている:Holdingsシートでティッカーを変更しても(例:列ヘッダーをリネーム)、他シートのクロスリファレンスは壊れない。Trade Logに行を追加してもワークブックの残りを無効化しない。

## よくあるフォローアッププロンプト

ベースワークブックを得たら、これらのプロンプトが最も価値を加える:

- *"リスクシートを追加 — 集中メトリクス、セクターエクスポージャー、保有銘柄間相関マトリクス。"*
- *"このポートフォリオに追加できる10ティッカーを、現在のテクニカルセットアップでランク付けしたウォッチリストシートを追加。"*
- *"マクロシートを追加 — VIX、10年利回り、ドル指数、原油 — ポートフォリオ挙動をマクロドライバーと相関させるために。"*
- *"indicators バケットラベルを数値スコア(−2〜+2)に変換するので、ポートフォリオレベルのシグナル平均を計算できる。"*
- *"異なるポートフォリオで同じワークブックを作成し、2つをマージして並列比較できる。"*

各プロンプトはフレッシュなワークブック生成をトリガー。

## 60秒でできないこと

正直な注意点:

- **カスタム数式アーキテクチャ。** 特定のExcel数式構造(特定の慣習を持つ名前付き範囲、ワークブック間参照、カスタムVBA)が必要なら、エクスポート上に手動で追加することになります。
- **リアルタイム更新。** ワークブックはエクスポート時のスナップショット。PickSkillは既に開いているExcelファイルにライブ更新をプッシュしません。リフレッシュするには、チャットから再エクスポート — 30秒。
- **重いマクロ/VBAスクリプト。** 出力はデータと数式。マクロ、カスタムリボン、条件付きVBAロジックは手動追加のまま。
- **ブローカー口座への直接接続。** PickSkillはサードパーティブローカーからライブポジションデータを取得しない;保有銘柄リストは[/portfolios](/portfolios)で手動で管理するポートフォリオから来る。

## 裏側でどう機能しているか

技術的に好奇心旺盛な人向け:

- PickSkillはまずワークブック構造を組み立てる(シート、列、データ値、数式のリスト)。
- 各シートのデータは[/indicators](/indicators)ダッシュボードを駆動するのと同じバックエンドロジックを使って生成。
- `.xlsx`ファイルはOpenXMLフォーマットを使って書き込まれる — すべてのセル、数式、条件付きフォーマット、スパークラインは本物のExcelオブジェクト。
- クロスシート参照は標準のA1記法を使うので、任意の互換スプレッドシートツールで動作。

出力は編集と共有目的で手作りワークブックのように振る舞いますが、秒で生成されます。

## FAQ

**出力を使うのにExcelインストールが必要?**
いいえ — `.xlsx`ファイルはExcel、Google Sheets、LibreOffice Calc、Apple Numbers、または任意のOpenXML互換ツールで開きます。すべての標準数式(SUM、AVERAGE、IF、INDEX、MATCH)はこれらのツール全体で動作;PickSkillはデフォルトエクスポートでExcel固有関数を避けます。

**ワークブックを開くと数式がライブ更新する?**
数式はワークブック自身のセルに対して更新(1つのセルの変更が依存セルに伝播)。新しい市場データをライブフェッチ*しない* — それにはアクティブなデータ接続が必要です。根底データをリフレッシュするには、チャットから再エクスポート。

**PickSkillアカウントを持たない人とワークブックを共有できる?**
はい — ワークブックはスタンドアロンファイル。一度ダウンロードすれば、通常Excelファイルを共有する方法(メール、クラウドドライブ、Slack)で共有。受信者は開いて使うのにPickSkillアカウントは必要ありません。

**A株や香港株保有銘柄を持つポートフォリオで動作する?**
はい。ワークブックは香港証券取引所ティッカー(`9988.HK`、`0700.HK`)とA株ティッカー(`600519.SS`、`000333.SZ`)を認識し、市場適切な慣習を適用。ストップ高日バー(A株)はSignal Trailシートでフラグされ、それらのバーからのテクニカルシグナルは外れ値として扱われる。

**このワークブックを毎週自動的に更新するにはどうする?**
2つのオプション。シンプルな方法:チャットセッションをブックマークしてエクスポートプロンプトを毎週再実行 — PickSkillが最新データでファイルを再構築。より自動化された方法(設計中):スケジュールでエクスポートを再実行し更新ファイルをメールするスケジュールワークフロー — 今後の機能として[workflows設計ドキュメント](/blog)を参照。

**独自のカスタム指標をワークブックに追加できる?**
Indicatorsシートは右側に独自のメトリクス用の列を追加できるように構造化されている。クロスシート参照は壊れません。PickSkillにメトリクスを計算してもらうにはチャットで依頼 — よくあるバリエーション(異なる指標期間、異なるバケット閾値、カスタムシグナル)はリクエストで追加可能。
