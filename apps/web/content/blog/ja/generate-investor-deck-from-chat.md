**投資家デッキは、リサーチ論題を10〜15スライドに蒸留し、別の投資家が5分で読んで意見を形成できるようにします。** スライドには明確な論題ステートメント、よく選ばれた数枚のチャート、評価サマリー、誠実なリスクセクションが必要。このチュートリアルはPickSkillリサーチ会話から直接そのデッキを生成する方法を説明します — 全スライドはあなたが既にチャットで行った作業に根ざし、全チャートに出典付き、`.pptx`が90秒でダウンロード準備完了。

これは5ステップのチュートリアル。最も長いステップは実際の投資判断の電話をするステップ。それ以外はすべて機械的。

### 重要なポイント

- **5ステップ、約90秒。** チャットを開く、論題を構築する、デッキを依頼する、プロンプトを編集する、ダウンロード。
- **デッキはチャット履歴から生成** — デッキ内の全チャートが、それが議論された会話の瞬間を引用。
- **デフォルト10〜15スライド。** カバー、論題、事業概要、財務、評価、テクニカルセットアップ、コンプス、シナリオ、リスク、付録。
- **出力は本物の`.pptx`ファイル** — 任意のスライドを編集、PowerPointまたはKeynoteからプレゼン、変換なしで共有。
- **米国・香港・A株銘柄で動作。** デッキは市場ごとに開示慣習を適応。

## なぜこれが重要か

投資家デッキフォーマットには理由があります:アナリストに弱いリンクを露呈する構造で論題を明確化させる。15ページのメモは「この銘柄が好き」と「ここに他の誰かが気にする理由がある」のギャップを隠せる。12スライドのデッキはそれらのギャップを即座に可視化し — そして、それを作る規律自体が、デッキを誰にも見せなくても根底分析を改善します。

3つの聴衆が恩恵を受けます:

- **潜在的な投資家やパートナー。** デッキはセカンドオピニオンのために論題を共有する最も効率的な方法。
- **正式な投資委員会。** 多くの投資クラブ、ファミリーオフィス、学生ファンドが新規ポジションにデッキ形式プレゼンを要求。
- **自分自身の分析的規律。** デッキを書くことがテスト;それを共有するのは任意。論題がデッキ化できないなら、論題には通常対処する価値のある隠れたギャップがある。

## 5ステップワークフロー

### ステップ1 — チャットを開き論題を構築

[/chat](/chat)へ移動し、リサーチをウォークスルー。典型的な論題構築会話は以下のように見える:

```text
TSMCに5年DCFを構築。WACC感応度を表示。
10-Kは粗利益軌道について何を教える?
ASMLとLam ResearchとEV/EBITDAとFCF成長で比較。
今のテクニカルセットアップ — MACD、RSI、MAスタック — はどう見える?
```

正確なプロンプトは*カバレッジ*ほど重要ではありません。会話の終わりまでに、以下を持つべき:

- 評価読み値(DCF、コンプス、または両方)
- 事業/ファンダメンタル視点(マージン軌道、成長ドライバー、貸借対照表)
- 競争ポジション(最も近い2〜3ピアに対して)
- テクニカル文脈(現在トレンドレジーム、モメンタム、主要レベル)

基礎をまだ構築していないなら、[60秒でDCFを構築](/blog/build-dcf-in-60-seconds)、[60秒で10-Kを要約](/blog/summarise-a-10k-in-60-seconds)、[テクニカル指標でポートフォリオを追跡](/blog/track-a-portfolio-with-indicators)で上流の会話を参照。

### ステップ2 — デッキを依頼

論題を構築したら、PickSkillにデッキ生成を依頼:

```text
この会話から12スライドの投資家デッキを生成。
論題を1文でリード。次に事業概要、財務、
評価(DCFとコンプス両方)、テクニカルセットアップ、シナリオ(強気/ベース/弱気)、
リスク、付録。全スライドをプレゼン準備完了に。
```

それが入力すべて。PickSkillは会話履歴をデッキのソース素材として使う — 全チャート、テーブル、箇条書きはチャット内の特定の交換に遡る。

### ステップ3 — 生成前に構造を洗練

PickSkillが生成にコミットする前に、提案されたスライドアウトラインが表示されます。この段階でよくある編集:

- **スライドの並び替え** — 「テクニカルセットアップをスライド4に、財務の前に移動」、トレーダー聴衆にプレゼンするなら。
- **スライドのカット** — 「コンプススライドをスキップ;聴衆は相対評価を気にしない」。
- **スライドの追加** — 「経営の質と資本配分実績についてのスライドを追加」。
- **深さの調整** — 「リスクセクションを2スライドに拡張 — 最初のスライドはファンダメンタルリスク、2つ目はテクニカル/市場構造リスク」。

アウトライン-その後-生成パターンは再生成サイクルを節約。完全なデッキは組み立てに30〜60秒、アウトラインの反復は5秒。

### ステップ4 — PickSkillがデッキを組み立てる間待つ

PickSkillは順番に以下を行います:

1. デッキに関連するチャット履歴をコンパイル(サイド会話、探索的な行き止まりをフィルター)。
2. 参照される各メトリクスの最新データを取得 — 価格、財務、倍率。
3. チャート(価格+テクニカル、評価コンプ、シナリオファン)を埋め込み画像としてレンダリング。
4. 投資家デッキ標準慣習でスライドコンテンツを生成:スライド2に論題ステートメント、評価フレームワークを明示、リスクは埋めるのではなくプレモータム。
5. フォーマット済みタイトル、ソースを引用する脚注、データ来歴のある最終付録を伴う`.pptx`を構成。

ストリーミングサマリーが表示されます。完了すると、7日間有効なダウンロードリンクを取得。

### ステップ5 — 任意のスライドを編集、プレゼン、反復

ダウンロードした`.pptx`はPowerPoint、Keynote、Google Slidesで開く。全スライドが完全に編集可能。

デッキレベルの編集には、チャットに戻る:

```text
スライド8(シナリオ)を再作成。強気ケースをよりアグレッシブに — 3年目
売上成長想定を15%ではなく25%に押し上げ。新しいDCF出力を表示。
```

```text
直近5年間の過去EV/EBITDA範囲と現在倍率がどこにあるかを示す
スライドを5と6の間に追加。
```

```text
このデッキの5分版を作成 — カバー、論題、評価サマリー、リスクのみ保持。
その他すべてをカット。
```

PickSkillは新しい構造で再生成。

> **今すぐ試す。** [チャットを開き](/chat)、論題をウォークスルーし、デッキを依頼。全ループは2分以内。

## 出力はどう見えるか

デフォルト12スライド構造:

| スライド | コンテンツ |
|---|---|
| **1. カバー** | ティッカー、企業名、プレゼンター、日付、1文の見出し |
| **2. 論題** | 30語以下の論題、価格目標/視点を1文、時間軸 |
| **3. 事業概要** | 売上分割(セグメント/地理)、競争ポジション、最近の戦略的動き |
| **4. 財務** | 直近4四半期の売上、EBITマージン、FCF、主要トレンドコールアウト |
| **5. 評価 — 絶対** | 主要想定を含むDCFサマリー、暗示価格、感応度グリッド |
| **6. 評価 — 相対** | EV/EBITDA、P/E、P/B 対セクター/ピア;現在 vs 過去範囲 |
| **7. テクニカルセットアップ** | MAスタック付き価格チャート、現在のMACD/RSI状態、主要サポート/レジスタンスレベル |
| **8. シナリオ** | 強気/ベース/弱気ケース — それぞれ明示的な想定変更と結果の価格目標 |
| **9. リスク — ファンダメンタル** | トップ3ファンダメンタルリスク(マージン圧縮、競争、規制、執行) |
| **10. リスク — 市場** | テクニカルリスク(買われすぎ、ダイバージェンス、主要レベル破壊)、流動性、ファクターエクスポージャー |
| **11. 結論** | アクション — 買い、保有、ウォッチ、回避 — サイジングガイダンスとエントリーレベル規律付き |
| **12. 付録** | データソース、方法論ノート、免責事項 |

スライドは一貫したタイポグラフィシステムとブランド整列カラーパレットを使用、会議設定での可読性のために設計(本文テキスト18pt以上、高コントラストチャート)。

## 90秒でできないこと

正直な注意点:

- **オリジナル一次リサーチ。** デッキはPickSkillのデータソースとチャット内の作業から構築。論題が一次リサーチインタビュー(顧客とのチャネルチェック、元従業員との会話)に依存するなら、そのコンテンツは手動追加が必要。
- **重くカスタムな評価モデル。** 標準DCF、相対評価、sum-of-the-partsフレームワークは箱から出して機能。バイオテックパイプラインのリアルオプションモデル、または料金ベースメカニクスを持つ規制公益事業DCFは、より多くのカスタム会話が上流で必要。
- **コンプライアンスグレードの法的開示。** 付録は基本的な免責事項を含む。規制された文脈(登録アドバイザー、ファンドマネージャー)でデッキを使用しているなら、コンプライアンス言語は弁護士のレビューが必要 — PickSkillは法的助言ではありません。

## FAQ

**ポートフォリオエクスポートPPTとどう違う?**
[ポートフォリオエクスポート](/blog/export-portfolio-to-powerpoint)は*データから*デッキを生成 — 保有銘柄、指標、評価スナップショットすべてが標準構造でレンダリング。このチュートリアルは*論題会話から*デッキを生成 — スライドコンテンツは固定テンプレートではなく実際にリサーチし議論したものに形作られる。定期的なポジションレビューにはポートフォリオエクスポートを、新規ポジションピッチにはチャットからデッキフローを使う。

**デッキはチャット内の特定の瞬間を引用する?**
はい — 付録の脚注とチャートキャプションは、コンテンツが生まれたプロンプト-応答の瞬間を参照。これによりスライドを擁護するときに分析チェーンをたどりやすく、作業をレビューする誰にとってもデッキが監査可能になります。

**他言語でデッキを生成できる?**
はい — ステップ2のプロンプトに「in [language]」を追加。PickSkillはプラットフォーム全体で使われる8ロケールをサポート。チャート、スライドタイトル、本文テキスト、脚注はすべて翻訳;データ値とティッカー記号はネイティブ形式のまま。

**論題が会話にないものに依存していたら?**
生成前に追加:「顧客YがQ3末までに製品Zに移行することを一次リサーチで別途確認済み」。PickSkillは関連スライドに帰属付き箇条書きとして追加を含めます。会話にないものは設計上デッキにない — デッキはチャットに根ざしています。

**ChatGPTやClaudeを直接使ってデッキを書くのと比較してどう?**
違いは*根拠付け*にあります。PickSkillのチャットは既にライブファイリングを取得、DCFを計算、指標を実行、コンプスを取得済み。デッキ構成はモデルの一般知識ではなく、それらの出典付きプリミティブに基づく。結果は捏造された数字や古いデータに対して実質的に脆弱でない — これが汎用チャットボット駆動のデッキ生成の失敗モード。

**デッキを保存して後で更新するために戻ってこれる?**
はい — チャットセッションは持続。会話を再オープンし「今日のデータでデッキを再生成し新しい決算リリースのためのスライドを追加」と依頼すると、PickSkillが新しい文脈を元の論題に重ねてデッキを再構築。
