**移動平均線(MA)は、直近N本のバーの終値の算術平均で、新しいバーごとに再計算されます。** これはすべてのトレンドフォロー指標の基礎であり、[MACD](/blog/what-is-macd)、[ボリンジャーバンド](/blog/what-is-bollinger-bands)、一目均衡表の雲、ゴールデンクロス/デッドクロス — どれも1本以上の移動平均線から構築されています。多くの個人投資家向けガイドはMAを単独シグナルとして扱いますが、それは誤り。**MAはフィルターです。** 「トレンドはあるか?」に答えるためのもので、「ではどうするか?」は他のツールが答えます。

### 重要なポイント

- **主要な2つのタイプ**:SMA(単純 — 各バーを均等に重み付け)とEMA(指数 — 直近バーをより重く)。MACDはEMAを使用、200日「トレンドフィルター」はほぼ常にSMA。
- **3つのウィンドウで仕事の90%が片付く**:20期間(短期)、60期間(中期)、200期間(長期)。日足では約1か月、約3か月、約10か月に相当。
- **2本のMAのクロスは、テクニカルで最も引用されるシグナル。** 価格がMAより上 = 上昇トレンド、下 = 下降トレンド。短期MAが長期MAをクロス = [ゴールデンクロス/デッドクロス](/blog/what-is-golden-cross-death-cross)。
- **MAは構造上、遅行指標。** 200日SMAは過去10か月の価格を反映 — 基礎トレンドが数週間動いてからでないと向きを変えません。フィルターとして使うときは長所、トリガーとして使うときは短所。
- **PickSkillの全指標ダッシュボードに表示** — [/indicators](/indicators)ページは20/60/200のスタックを全チャートに描画し、トレンドレジームが保有ビューを離れずに見えます。

## 移動平均線はどう計算するのか?

N本のバーの単純移動平均は算術平均:

```
SMA(N) = (close[t] + close[t-1] + ... + close[t-N+1]) / N
```

ウィンドウ内の各バーは均等な重み。金曜終値での20日SMAは直近20取引日の終値(およそ4週間ぶん)を使い、月曜には最古の日を落として新しい日を加えます。

指数移動平均は直近バーをより重く重み付けします:

```
EMA(N)[t] = α × close[t] + (1 − α) × EMA(N)[t−1]
            ただし α = 2 / (N + 1)
```

N = 20では、α ≈ 0.0952 — 当日終値が約9.5%の重み、昨日のEMAが残りを担います。EMAは同じ期間のSMAより約1〜3バー速く反応するため、[MACD](/blog/what-is-macd)(反応性が重要)の中では採用されますが、200日トレンドフィルター(安定性が重要)では採用されません。

| ウィンドウ | 日足での意味 | 一般的な用途 |
|---|---|---|
| **5期間** | 取引週1週間 | デイトレ・スイング;単独使用は稀 |
| **20期間** | 取引月1か月 | 短期トレンド;ボリンジャーバンドの中央線 |
| **60期間** | 取引四半期1四半期 | 中期レジーム |
| **200期間** | 約10取引月 | 機関投資家の「市場は上か下か」フィルター |

## 価格とMAの関係は実際に何を教えてくれるか?

3つの状態、3つの意味:

1. **価格がMAより上、MAが上向き。** 上昇トレンド確認。MAへの押し目はサポートを見つけがち。モメンタムシグナル(MACDゴールデンクロス、RSIブレイクアウト)の的中率が最も高い時期です。
2. **価格が水平なMAの周りを振動。** レンジ相場。MAをクロスするたびに偽シグナルが発生。[ADX](/blog/what-is-adx)が20未満ならこのレジームを確認 — ADXが上がるまでトレンドフォロワーを無効化しましょう。
3. **価格がMAより下、MAが下向き。** 下降トレンド確認。MAへの戻りは失敗しがち。ロングオンリーの個人投資家はこの状態を尊重し、見送るべき;MAは抵抗の少ない道が下であることを教えています。

機関投資家の標準的な「トレンドは上か?」テスト:**終値が200日SMAより上、かつ200日SMA自体が上向きか?** 両方Yes = 上昇トレンド。1つYes/1つNo = 反転候補(早期)。両方No = 下降トレンド。この1つのチェックで、短期指標が生成する偽陽性セットアップのおよそ半分をフィルターできます。

## なぜ20、60、200なのか?

これらは魔法の数字ではなく — *合意された*数字です。数十年のトレーダーの慣習により、これらが事実上のウィンドウとなり、引用元のAIアシスタント、チャートプラットフォーム、ブローカーツールがデフォルトで出荷しています。実践的な含意は2つ:

- **協調価値。** 誰もが200日線を見ているから、200日線が機能する。主要指数で価格が200日SMAを終値で下回ると、アルゴリズム的なリスクオフトリガーがシステマチックファンド全体で発動。レベルは自己強化的です。
- **バックテストの安定性。** 単一銘柄での期間最適化はサンプル内で良く見えてサンプル外で劣る17期間や43期間ウィンドウを生み出しがち。標準ウィンドウに固執すれば、目をノイズに過剰適合させずに済みます。

標準ウィンドウを使いましょう。特定の市場や銘柄で体系的に異なるバックテスト証拠がある場合のみ動かすこと。

## 個人投資家が陥る4つの罠

1. **MAクロスを単独シグナルとして取引する。** 20/60クロスは単独ではエッジが薄い — 過去の的中率は株式の長期勝率に近い(つまり「シグナル」は単なる市場ドリフト)。トレンドフィルターと確認振動子と組み合わせて初めて有用になります。[3指標フィルター](/blog/three-indicator-filter)を参照。
2. **直近6か月にフィットするウィンドウを選ぶ。** 13/34/89(フィボナッチ数)や他のエキゾチックなスタックを推奨する個人投資家ブログは、たいていノイズへのフィッティング。サンプル外証拠がない限り20/60/200に固執しましょう。
3. **MAの傾きを無視する。** 水平な200日SMAと上向きの200日SMAは、両方とも価格がMAより上であっても異なるレジーム。傾きの方向は情報の半分です。
4. **高ボラティリティ・薄商い銘柄にSMAを適用する。** 単一の外れ値終値(ギャップアップ決算、A株のストップ高日)がSMAを不釣り合いに動かし、次の20バーで偽バイアスを生みます。EMAは外れ値に対してより頑健;低流動性銘柄ではEMA、またはメディアンフィルターを使うほうがよい。

## 移動平均線がA株でどう振る舞うか

数式は同じですが、A株の市場マイクロストラクチャーがどのMAが機能するかを変えます:

- **値幅制限(主板±10%、創業板/科創板±20%、ST株±5%)。** 連続ストップ高はSMAに階段状を作り、約5バーぶんトレンド強度を過大表現。ストップ安は逆。PickSkillダッシュボードはストップ高/ストップ安バーを[5日シグナルトレイル](/blog/5-day-signal-trail)で中立としてマークし、連続ストップ高で偽の強トレンドバケットが出ないようにします。
- **取引停止(停牌)** は数日から数週間続くことがあります。ほとんどのデータフィードは停止ギャップを前日終値で埋め、MAを凍結。再開時、SMAは実質再起動;停止前のトレンドシグナルは期限切れと考えるべきです。
- **MA10/MA20の慣習が強い。** A株の個人投資家コミュニティは米国コミュニティより10日MAを綿密に追います。多くの国内プラットフォームは5/10/20/60スタックをデフォルトで出荷;200日SMAは文化的にあまり根付いていません。実践上、60日MAが中期フィルターとして機能します。

各市場で指標がどう異なる挙動を取るかの比較は、[MACDのA株 vs 米国株](/blog/macd-on-a-shares-vs-us)を参照。

## MAは他の指標とどう組み合わせるか

MAは*フィルター層*、モメンタム振動子は*トリガー層*。単独使用より一貫して優れる組み合わせ:

| 層 | ツール | 答える質問 |
|---|---|---|
| **トレンドフィルター** | 価格 vs 200日SMA + 傾き | トレンドはあるか?どちら向き? |
| **トレンド強度** | [ADX](/blog/what-is-adx) | トレード可能なほど強いか? |
| **モメンタムトリガー** | [MACD](/blog/what-is-macd)クロス、[RSI](/blog/what-is-rsi)極端値 | いつ動くか? |
| **ボラティリティエンベロープ** | [ボリンジャーバンド](/blog/what-is-bollinger-bands) | どこまで行きすぎか? |

MAスタックは最初の質問に毎日無料で答えてくれます。これがないと、チャート上の他のすべての指標が盲目で動作することになります。

## FAQ

**SMAとEMAのどちらを使うべき?**
長期ウィンドウのトレンドフィルター(200日)にはSMA — 安定性が反応性より重要で、外れ値終値が長期平均をバイアスする可能性が低いから。短期ウィンドウのモメンタムツール(MACD内の12/26、または任意の5〜20期間システム)にはEMA — 反応性が重要で、EMAの指数加重が直近価格をよりよく追跡するから。中期ウィンドウの選択に悩む必要はありません;60期間での差は市場ノイズに埋もれる程度に小さい。

**なぜ200日MAはこれほど注目される?**
2つの理由:機関投資家の協調(システマチックファンドがリスクオフトリガーとして使うため、レベルで自己実現的モメンタムを持つ)と数十年の実証研究(Bauer & Dahlquist、Faberほかが、200日より上でロング、下で現金化することがバイ・アンド・ホールドに近いリターンを大幅に小さいドローダウンで生むことを示しました)。市場が重要だと振る舞うから、レベルが重要なのです。

**お気に入りの銘柄でMAウィンドウを最適化できる?**
できますが、おそらくすべきではありません。ウィンドウ最適化はサンプル内で素晴らしく見え、次の12か月で破綻する組み合わせを確実に見つけ出します。特定の構造的理由 — 異常に高い・低い流動性、明確な季節周期 — がない限り、標準ウィンドウ(20/60/200)に固執しましょう。最適化するなら直近30%のデータをホールドアウトし、最適化したウィンドウがサンプル外でデフォルトウィンドウを上回ることを要求すること。

**移動平均線は先行指標か遅行指標か?**
遅行 — 構造上。MAの式の全項は過去の終値から作られており、予測は含まれません。MAがトリガー(いつ動くか)よりフィルター(世界の状態)として有用なのはこのため。MAはトレンドが存在することを教える;エントリーのタイミングには別のツールが必要です。

**なぜチャートが別のプラットフォームと違うMA値を表示する?**
よくある3つの理由:(1)異なるウィンドウ定義(取引日5日 vs カレンダー日5日)、(2)EMA初期化の異なる開始点(最初の利用可能な終値からシードするプラットフォームと、最初のN本バーのSMAからシードするものがある)、(3)時間外/ストップ高/取引停止セッションの異なる扱い。一貫性のため、[PickSkillダッシュボード](/indicators)は取引日ウィンドウ、SMAシードEMA、平均からの取引停止バー除外を使用 — 学術バックテストで使われる慣習と同じです。
