**포트폴리오 분석으로 진짜 PowerPoint 덱을 만들려면 예전에는 Excel에서 차트를 뽑는 데 한 시간, PowerPoint에서 슬라이드를 배치하는 데 한 시간, 누군가 검토한 후 서식을 고치는 데 또 한 시간이 걸렸습니다.** 이 튜토리얼은 같은 워크플로우를 60초 만에 보여줍니다 — 모든 차트가 실시간 데이터에서 출처, 모든 슬라이드가 전문 리서치 데스크의 디자인 관행으로 배치, 그리고 다운로드, 편집, 프레젠테이션 준비가 끝난 `.pptx` 파일. 분석의 수학은 직접 만든 것과 동일합니다 — 바뀌는 것은 조립에 드는 시간과 발표에 드는 시간의 비율입니다.

이 튜토리얼은 4단계입니다. 각 단계는 하나의 프롬프트 또는 하나의 클릭입니다. PickSkill에 이미 포트폴리오가 설정되어 있다면 전체 흐름을 1분 안에 실행할 수 있습니다.

### 핵심 요약

- **4단계, 약 60초.** 포트폴리오를 열고, 내보내기 클릭, PPT 선택, 슬라이드 제목 편집.
- **모든 차트가 실시간 데이터에서 출처.** 지표는 최신 종가로 실행되고, 재무는 최신 공시에서 가져오며, 가치 평가 배수는 현재 컨센서스를 반영합니다.
- **PPTX는 진짜 PowerPoint 파일입니다** — 스크린샷도, PDF도 아닙니다. 열고, 어느 슬라이드든 편집하고, PowerPoint 또는 Keynote에서 발표하세요.
- **미국, 홍콩, A주 보유 종목 전반에 작동.** 덱은 포트폴리오에 포함된 시장에 맞춰 제목과 애널리스트 어휘를 조정합니다.
- **출력에는 기본 10슬라이드 구조가 포함됩니다** — 표지, 보유 종목 요약, 기술적 신호, 가치 평가 스냅샷, 종목별 지표 상세, 리스크 요약, 부록.

## 왜 중요한가

대부분의 개인 투자자가 자신의 분석을 공유하지 않는 이유는 *프레젠테이션 계층*이 마찰이기 때문입니다. 공유하고 싶은 분석을 만들고 나면, 그것을 공유 가능하게 만드는 덱 조립에 인내심이 바닥납니다. 결과: 다른 사람의 눈에 의해 압력을 받지 못하는 많은 사적인 좋은 작업.

PickSkill은 덱 조립 단계를 한 번의 클릭으로 압축하여 시간이 분석으로 돌아가게 합니다. 세 가지 청중이 이득을 봅니다:

- **당신의 미래 자신.** 발표 가능한 산출물에 입장(stake)을 저장하면 3개월 후 돌아와 읽을 수 있는 기록이 생깁니다.
- **포지션을 논의하는 누군가.** 친구, 투자 클럽, 가족 자금을 다루는 파트너. 덱은 대화를 채팅 스레드보다 더 구조화된 것으로 만듭니다.
- **잠재적 상사나 면접 위원회.** 지망 애널리스트들에게: 실제 포트폴리오에 대한 깔끔한 덱은 면접에 가져갈 수 있는 가장 신뢰할 만한 도구입니다.

## 4단계 워크플로우

### 1단계 — 내보낼 포트폴리오 열기

[/portfolios](/portfolios)로 이동. 덱으로 만들 포트폴리오를 선택하거나, 3~10개 종목을 추가해 새로 만드세요. (첫 설정은 [Track a Portfolio with Indicators](/blog/track-a-portfolio-with-indicators)를 참고하세요.)

덱은 보유 종목 3~15개 범위의 포트폴리오에 맞게 설계되었습니다. 3개 미만이면 덱이 얇고, 15개 초과면 종목별 섹션이 일반적인 20분 프레젠테이션에 너무 길어집니다.

### 2단계 — "Export to PowerPoint" 클릭

포트폴리오 상세 페이지 헤더에 "Export to chat" 버튼 클러스터가 있습니다. PowerPoint 버튼을 클릭하세요. PickSkill이 채팅을 열고 포트폴리오의 현재 상태 — 보유 종목, 최신 지표 수치, 최근 신호 변화, 다이버전스 표시 — 가 포함된 사전 작성 프롬프트를 제공합니다.

보내기 전에 프롬프트를 편집할 수 있습니다. 두 가지 흔한 편집:

- **로케일 변경**: 덱 제목과 챕터 헤딩은 계정 언어를 기본으로 합니다. 덱이 특정 청중을 위한 것이라면(미국 팀을 위한 영어 발표, 본토 독자를 위한 중국어 발표) 그에 맞게 조정하세요.
- **커스텀 강조 추가**: "기술적 섹션을 다이버전스와 과매수 상태에 집중" 또는 "FCF 내러티브로 시작" — PickSkill이 덱 구성을 당신의 강조 쪽으로 가중합니다.

### 3단계 — PickSkill이 덱을 조립하는 동안 약 30초 대기

PickSkill은 순서대로 다음을 수행합니다:

1. 모든 보유 종목에 대해 현재 가격, 최신 종가, 6개월 가격 이력을 가져옵니다.
2. 각 보유 종목에 대해 전체 지표 묶음([MACD](/blog/what-is-macd), [RSI](/blog/what-is-rsi), [KDJ](/blog/what-is-kdj), [볼린저 밴드](/blog/what-is-bollinger-bands), [ADX](/blog/what-is-adx), MA 묶음, [거래량](/blog/what-is-volume-analysis), [자금흐름](/blog/what-is-capital-flow))을 실행합니다.
3. 활성 신호 패턴(골든크로스, 다이버전스, 지지/저항 근접, 5일 윈도우 동안의 버킷-추세 변화)을 탐지합니다.
4. 가치 평가 스냅샷(P/E, EV/EBITDA, P/B)과 최근 재무 요약을 가져옵니다.
5. 지표 차트를 임베드된 이미지로 렌더링합니다.
6. 편집 리서치 발표를 위한 디자인 관행으로 덱 콘텐츠를 생성합니다.
7. 임베드된 차트, 서식 있는 테이블, 편집 가능한 슬라이드 제목으로 `.pptx` 파일을 구성합니다.

작업이 진행되는 동안 스트리밍 요약을 볼 수 있습니다. 완료되면 7일간 유효한 다운로드 링크를 받습니다.

### 4단계 — 어떤 슬라이드든 편집, 특정 섹션 재실행

다운로드된 `.pptx`는 진짜 PowerPoint 파일입니다. PowerPoint, Keynote, 또는 Google Slides에서 여세요. 모든 슬라이드 제목, 차트 캡션, 테이블이 직접 편집 가능합니다.

더 깊은 편집을 위해서는 채팅으로 돌아가 특정 변경을 요청하세요:

```text
Re-do the technical signals slide. Highlight the three holdings with active 
divergence and demote the rest to a single summary bullet.
```

```text
Add a slide on capital flow trends. For each holding, show whether net flow 
has been positive or negative over the last 5 sessions.
```

```text
Compress the per-holding detail slides — one slide per holding instead of two, 
showing only the technical bucket trail and the latest indicator chart.
```

PickSkill이 새로운 강조로 덱 조립을 재실행하고 새 다운로드를 제공합니다.

> **지금 해 보세요.** [/portfolios로 이동](/portfolios)하여 어떤 포트폴리오든 클릭하고 "Export to PowerPoint"를 클릭하세요. 전체 루프는 1분 안에 끝납니다.

## 출력이 어떻게 보이는가

기본 10슬라이드 구조:

| 슬라이드 | 내용 |
|---|---|
| **1. 표지** | 포트폴리오 이름, 날짜, 보유 종목 수, 총 가치, 헤드라인 성과 |
| **2. 보유 종목 요약** | 모든 보유 종목의 테이블 — 티커, 비중, 현재 가격, 1일/5일/1개월 수익률, 시장 |
| **3. 기술적 신호 개요** | 각 보유 종목의 8개 지표 차원별 버킷을 보여주는 히트맵 형식 테이블 |
| **4. 활성 알림** | 중요 신호 변화가 있는 보유 종목 — 신선한 골든크로스, 다이버전스 표시, 과매도 수치 |
| **5. 가치 평가 스냅샷** | 보유 종목 전반의 P/E, P/B, EV/EBITDA 대 섹터 중앙값 |
| **6–8. 종목별 상세** | 각 보유 종목(또는 포트폴리오가 큰 경우 상위 비중 종목)에 대해: 주요 MA가 있는 가격 차트, 지배적 신호 내러티브, 5일 버킷 트레일 |
| **9. 리스크 요약** | 집중 위험, 섹터 편향, 다이버전스 경고, 핵심 지지/저항에 접근하는 보유 종목 |
| **10. 부록** | 데이터 출처, 지표 정의, 타임스탬프, 면책조항 |

슬라이드는 일관된 타이포그래피 시스템, 브랜드 정렬 컬러 팔레트, 임베드된 차트(차트의 텍스트 설명이 아님)를 사용합니다. 결과는 슬라이드별 서식 작업 없이 발표 준비가 된 상태입니다.

## 흔한 후속 프롬프트

기본 덱을 만든 후, 이 프롬프트들이 효과를 냅니다:

- *"가장 큰 포지션에 대해 1슬라이드 강세론과 1슬라이드 약세론을 추가해 주세요."*
- *"기술적 신호 개요를 신호등 형식 — 녹색 / 황색 / 적색 점만 — 으로 바꿔 주세요."*
- *"각 보유 종목의 전체 5일 버킷 이력을 보여주는 부록 슬라이드를 추가해 주세요."*
- *"중국어 청중을 위해 이 덱의 만다린 버전을 만들어 주세요."*
- *"슬라이드보다 산문을 선호하는 사람을 위해 같은 내용을 Word 문서로 생성해 주세요."* ([형식 트레이드오프는 Generate an Investor Deck from a Chat](/blog/generate-investor-deck-from-chat)를 참고하세요.)

각 프롬프트는 새 구조로 새로운 덱 생성을 트리거합니다.

## 60초 안에 할 수 없는 것

정직한 주의사항:

- **커스텀 디자인 템플릿.** 덱은 PickSkill 자체 디자인 템플릿을 사용합니다. 슬라이드 콘텐츠는 자유롭게 편집할 수 있지만, 무거운 브랜딩의 기업 템플릿(커스텀 폰트, 로고 배치, 특정 컬러 시스템)은 내보내기 후 수동 설정이 필요합니다. 대부분의 PowerPoint 테마는 디자인 → 변형을 통해 내보낸 덱에 깔끔하게 적용됩니다.
- **예측 / 전망 슬라이드.** 덱은 *현재* 상태와 *최근* 신호를 보여줍니다 — 가격이 다음에 어디로 갈지 예측하지 않습니다. 예측은 명시적 프롬프트와 추가 맥락을 필요로 합니다.
- **M&A를 위한 시너지 모델링.** 거래 피치 맥락에서 덱을 사용한다면, 종목별 슬라이드는 독립적인 경우를 다루며 거래 특정 시너지 모델링은 그 자체의 대화가 필요합니다.

## 내부에서 어떻게 작동하는가

기술적으로 호기심이 있는 분들을 위해:

- PickSkill은 먼저 슬라이드 구조를 컴파일합니다(차트 사양, 텍스트 콘텐츠, 레이아웃 힌트가 있는 JSON 골격).
- 각 차트는 [/indicators](/indicators) 대시보드를 구동하는 동일한 지표 코드를 사용해 고해상도 PNG로 렌더링됩니다.
- PPTX 파일은 OpenXML 형식을 사용해 프로그램적으로 조립됩니다 — 모든 도형, 테이블, 이미지가 진짜 PowerPoint 객체이지 평면 래스터화가 아닙니다.
- 슬라이드 제목과 본문은 모델의 분석 출력에 바인딩되어 있어, 채팅에서 편집하면 분석이 재실행되고 새 덱이 생성됩니다.

결과는 편집 목적으로 손으로 만든 덱처럼 작동하지만 몇 초 만에 생성됩니다.

## FAQ

**출력을 사용하려면 PowerPoint를 설치해야 하나요?**
아니요 — `.pptx` 파일은 PowerPoint, Keynote, Google Slides, LibreOffice Impress, 또는 어떤 PPTX 호환 도구에서도 열립니다. PowerPoint가 없어도 모든 현대 Office 호환 스위트가 파일을 올바르게 렌더링합니다.

**덱 안의 개별 차트를 편집할 수 있나요?**
차트는 이미지로 임베드되어 있습니다 — 이동, 크기 조정, 교체는 가능하지만 PowerPoint 내에서 직접 다시 스타일링은 불가능합니다. 차트를 바꾸려면 채팅으로 돌아가 새 스타일(다른 시간대, 다른 지표 등)로 재렌더링을 요청하세요. 새 차트가 다음 덱 내보내기에 임베드됩니다.

**A주 또는 홍콩 보유 종목이 있는 포트폴리오에서도 작동하나요?**
네. PickSkill은 HKEx 티커(예: `9988.HK`, `0700.HK`)와 A주 티커(`600519.SS`, `000333.SZ`)를 인식하고 시장별로 올바른 데이터 출처를 가져옵니다. 덱은 적응합니다: A주 보유 종목은 [MACD-on-A-shares](/blog/macd-on-a-shares-vs-us) 해석 오버레이를 받고, 상한가 봉은 기술적 섹션에서 이상치로 표시됩니다.

**다운로드 링크는 얼마나 오래 유효한가요?**
7일입니다. 파일 자체는 다운로드되면 영구적입니다 — 새로 고친 버전이 필요하면 언제든 채팅에서 다시 생성하세요. 다운로드 URL은 계정에 범위가 한정되므로, 계정 외부 누군가와 URL을 공유해도 작동하지 않습니다.

**같은 덱을 다른 언어로 생성할 수 있나요?**
