개인 투자자는 같은 벽을 만납니다: 8–15 개 종목을 보유, 어떤 종목이 기술적 변곡점에 있는지 알고 싶지만 매일 모든 종목의 차트를 읽는 건 지속 불가능. 이 튜토리얼은 PickSkill 에서 포트폴리오를 설정하고, 8 개 지표 차원을 전 종목에 적용하고, 한눈에 보는 신호 대시보드를 받아 정말 주목할 가치 있는 종목에만 줌인하는 방법.

### 핵심 요약

- **8 개 지표 차원** 기본 탑재: 가격·펀더멘털·센티먼트·자금 흐름·다이버전스·KDJ·MACD·지지/저항.
- **하나의 대시보드, 모든 종목.** 각 지표 차원이 모든 보유를 다루는 정렬 가능한 HTML 리포트.
- **요청 시 갱신** 또는 주기 일정.
- **미국·홍콩·A 주 지원.**
- **기존 [/portfolios](/portfolios)** 보유 저장 + [/indicators](/indicators) 대시보드 활용 — 채팅이 새 뷰 요청 수단.

## 왜 중요한가

8 차원에서 12 종목 수동 추적 = 매일 96 회 차트 조회. 대부분 "걱정되는 몇 종목만 본다"로 압축하지만, 그건 걱정 안 되는 종목에서 오는 신호가 조용히 새어 나간다는 뜻. 대시보드는 96 회 조회를 한 번의 스캔으로: 12 종목 × 8 차원을 한 페이지에 정렬, 변곡점 하이라이트.

## 5 단계 워크플로

### Step 1 — 포트폴리오 생성 또는 선택

[/portfolios](/portfolios) 에서 신규 생성 또는 기존 선택. 여러 포트폴리오 가능 — 미국 대형주, A 주, 실험적 바스켓.

자연어로 보유를 추가했다면("내 메인 포트에 NVDA 100 주 $135 추가"), 이미 거기 있음.

### Step 2 — 관심 지표 차원 선택

| 차원 | 답하는 질문 |
|---|---|
| **가격** | 각 종목이 5/20/60 일 이동평균 대비 어디? |
| **펀더멘털** | 각 보유의 PER/PBR/배당수익률/FCF 수익률 |
| **센티먼트** | 각 종목의 뉴스 + 애널리스트 액션 |
| **자금 흐름** | 지난 5–20 거래일의 순기관 흐름 |
| **다이버전스** | 가격 vs 거래량, 가격 vs RSI 다이버전스 — 조기 경고 |
| **KDJ** | 스토캐스틱 과매도(J<0)/과매수(J>100) |
| **MACD** | 골든크로스/데드크로스 + 히스토그램 모멘텀 |
| **지지/저항** | 손실 포지션, 저항 시험 중 |

대부분 개인 투자자 워크플로의 가장 레버리지 큰 넷: 가격, MACD, 자금 흐름, 다이버전스.

### Step 3 — 채팅 열어 대시보드 요청

```text
내 "테크 대형주" 포트폴리오에서:
- MACD 대시보드 실행
- 오늘 골든크로스 또는 데드크로스 포지션 강조
- 히스토그램 강도로 정렬
```

PickSkill 이 포트폴리오에서 보유 목록을 가져와, 각 종목에 대해 최신 일봉 기준으로 MACD 계산, 다운로드 가능한 HTML 대시보드 생성, 변곡 종목을 인라인 제시.

### Step 4 — 다른 차원과 교차 참조

단일 차원도 유용하지만 신호가 수렴할 때 강해짐. MACD 대시보드 후:

```text
골든크로스로 표시한 종목 중 다음도 해당하는 것:
- 지난 5 일 자금 흐름이 양?
- 강세 다이버전스(가격 하락, RSI 상승)?
- 60 일 이동평균 위?
```

이게 실제 리서치 워크플로 — 12 후보에서 2–3 개로 신호 누적해 좁힘.

### Step 5 — 쓸 만한 대시보드 일정화

```text
"테크 대형주"의 MACD 대시보드를 매 영업일 미국 종가 후 실행.
1 종목 이상 크로스했을 때만 이메일(매일 노이즈 이메일은 아님).
```

조건부 이메일 패턴이 유용한 도구와 또 하나의 받은편지함 소스의 차이.

> **지금 시도.** [포트폴리오 만들기](/portfolios) 후 [채팅 열기](/chat) 에서 Step 3 프롬프트 붙여넣기.

## 자세히 알 가치 있는 지표

- **MACD** — 12 기간 EMA 와 26 기간 EMA 의 교차, 9 기간 신호선. 골든크로스/데드크로스 + 히스토그램 모멘텀.
- **KDJ** — 스토캐스틱 변형, J 라인이 극값을 증폭. J<0 = 과매도; J>100 = 과매수.
- **다이버전스** — 가격은 신고가인데 RSI 는 아님(약세); 가격은 신저가인데 RSI 는 아님(강세). 종종 반전의 가장 이른 신호.
- **자금 흐름** — 순기관/대량 주문 흐름. 홍콩/A 주는 거래소 공시 데이터, 미국은 대량 거래 데이터 근사.

## 이 방식으로 못 하는 것

- **장중 신호.** 대시보드는 기본 일봉 지표.
- **옵션 전략.** 지표는 주식만.
- **자동 주문.** PickSkill 은 신호 표면화, 주문 안 함.

## 기초 클러스터와의 상호 보완

기초 클러스터([DCF](/blog/what-is-dcf)·[WACC](/blog/what-is-wacc)·[FCF](/blog/what-is-fcf)·[10-K](/blog/how-to-read-10k))는 **펀더멘털 리서치**; 본 튜토리얼은 **기술적 모니터링**. 대부분 개인 투자자 워크플로는 두 층 모두에서 이익 — 펀더로 종목 선정, 테크니컬로 타이밍.

## FAQ

**포트폴리오를 수동 설정해야 하나?**
아니요 — 자연어로 보유 추가 가능, [/portfolios](/portfolios) 에 동기화. CSV 업로드도 가능.

**데이터 얼마나 신선?**
미국은 종가, A 주와 홍콩은 장중 실시간. 자금 흐름은 모든 시장 일별 마감.

**여러 포트폴리오 동시 실행 가능?**
가능 — "모든 내 포트폴리오에서 MACD 대시보드 실행, 포트폴리오별로 그룹화".

**A 주에서 작동?**
작동 — 엔진이 A 주 티커와 거래일 캘린더를 올바르게 처리.

**스케줄 이메일 설정?**
채팅에서 일정화 ("매 영업일 미국 종가 후 MACD 대시보드 실행, ≥1 크로스면 이메일"). 가입 이메일과 통합.
